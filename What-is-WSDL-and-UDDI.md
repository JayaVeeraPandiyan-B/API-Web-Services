WSDL and UDDI rendumey XML based technologies that were a part of the original "Big Web Services" architecture often used in conjuction with SOAP

WSDL: Service provider web service ku interface ah publish pannum athula web service ku thevaiyana ella attributes um irukkkum. Ithu XML based interface and is called
"Web Service Description Language" - WSDL. ithu XML based interface Web Service la irukura functionalities ah describe panna use aaguthu.

Ippo oru service provider client ku request/Web services client eh direct ah therinji iruntha WSDL doc use panni kudukalam

But Service provider and Client rendu perkum yaruney theriyala na appo tha UDDI varuthu athuvum WSDL moolama tha.

UDDI: Oru web service provider Web Service ah online directoty la publish pandranga, anga consumers Web Service ah query and search panalam. This online registry/directory
is called UDDI (Universal Description, Discovery, and Integration). WSDL eh online la pota UDDI so that customers search panni pathupanga.
